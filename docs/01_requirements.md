# 要件整理

## 目的

自社が運用する複数のInstagramアカウントについて、事務スタッフがAI社員に自然文で指示し、投稿企画、文章作成、画像構成、リール台本、投稿管理、承認、実績入力、分析改善までを一つの社内Webシステムで進められるようにする。

課金、契約、料金プラン、顧客企業ごとのテナント管理は対象外とする。

## 管理単位

管理単位は「案件」とする。各案件にInstagramアカウント、会社・店舗情報、商品・サービス、ターゲット、ブランドルール、AI社員、投稿、素材、ナレッジ、実績を紐付ける。

## 利用者と権限概要

| 利用者 | 主な役割 |
| --- | --- |
| システム管理者 | ユーザー、権限、案件、AI社員、ブランド、投稿、分析、監査ログを管理する |
| 事務スタッフ | AI社員へ指示し、投稿案作成、保存、タスク化、素材登録、実績入力を行う |
| 確認者・承認者 | 投稿内容を確認し、コメント、修正依頼、承認、却下を行う |
| 閲覧者 | 投稿、進捗、カレンダー、分析レポートを閲覧する |

## 利用者別業務フロー

### システム管理者

1. ログインする
2. ユーザーを登録する
3. ユーザーに全体ロールを設定する
4. 案件を作成する
5. 案件メンバーと案件内ロールを設定する
6. Instagramアカウント、ブランド、ターゲット、商品・サービス、投稿ルールを登録する
7. AI社員を案件へ紐付ける
8. 操作履歴と分析状況を確認する

### 事務スタッフ

1. ログインする
2. 担当案件を選択する
3. AI社員チャットで投稿企画や投稿案を依頼する
4. AI生成結果を確認する
5. 必要に応じて修正指示を出す
6. 投稿案として保存する
7. 投稿予定日、担当者、必要素材、タスクを設定する
8. 確認者または承認者へ依頼する
9. 投稿後に実績数値を入力する
10. AI分析を実行し、次回企画へ反映する

### 確認者・承認者

1. ログインする
2. 確認待ちまたは承認待ち一覧を見る
3. 投稿詳細を確認する
4. コメント、修正依頼、承認、却下を行う
5. 修正履歴と承認履歴を確認する

### 閲覧者

1. ログインする
2. 許可された案件の投稿一覧、カレンダー、分析を見る

## ユーザーフロー

### 初期設定フロー

ログイン → 案件作成 → 案件メンバー設定 → Instagramアカウント登録 → ブランド設定 → ターゲット設定 → 商品・サービス登録 → AI社員設定 → ナレッジ登録

### 投稿作成フロー

ログイン → AI社員チャット → 案件選択 → 指示入力 → AI生成 → 内容確認 → 修正または保存 → 投稿詳細作成 → タスク作成 → 確認依頼

### 承認フロー

確認依頼通知 → 投稿詳細確認 → コメントまたは修正依頼 → 再提出 → 承認 → 投稿予約済みに変更

### 分析フロー

投稿済み一覧 → 実績入力 → 指標集計 → AI分析 → 改善提案保存 → 次月企画作成
