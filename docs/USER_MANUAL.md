# Instagram運用AI社員システム 操作マニュアル

## 1. 利用URL

本番環境はVercelで公開されています。

```text
https://instaai-six.vercel.app
```

このURLは、PCを閉じていても利用できます。アプリ本体はVercel、データ保存はSupabase、AI生成はOpenAI APIで動きます。

## 2. ログイン

1. `https://instaai-six.vercel.app/login` を開きます。
2. 登録済みのメールアドレスを入力します。
3. 登録済みのパスワードを入力します。
4. `ログイン` を押します。

ログインできない場合は、Supabaseの `Authentication > Users` に対象メールアドレスが登録されているか確認してください。

## 3. ユーザー追加

他の人が使う場合は、管理者がSupabaseでユーザーを追加します。

1. Supabaseを開きます。
2. 対象プロジェクトを開きます。
3. `Authentication` を開きます。
4. `Users` を開きます。
5. `Add user` を押します。
6. メールアドレスとパスワードを設定します。
7. 作成したメールアドレスとパスワードを利用者に共有します。

利用者はVercelの公開URLからログインできます。

## 4. 基本の運用手順

日々の運用は、以下の順番で進めます。

1. `案件一覧` で案件を登録します。
2. 案件詳細でブランド情報、ターゲット、AI社員設定を入力します。
3. `AI社員チャット` で投稿企画や投稿文を作成します。
4. AIの返信から `投稿案として保存` を押します。
5. `投稿一覧` で投稿案を編集します。
6. 投稿詳細でステータス、投稿予定日時、タスク、画像生成を管理します。
7. `承認フロー` で承認待ち投稿を確認します。
8. 投稿後に `投稿実績入力` でInstagramインサイト数値を入力します。
9. `月次レポート` で改善提案と翌月方針を生成します。

## 5. 画面別の使い方

### ダッシュボード

ログイン後の最初の画面です。案件や投稿の状況を確認します。

### 案件一覧

案件を登録・確認する画面です。

1. 左メニューの `案件一覧` を押します。
2. 案件名、会社名、業種などを入力します。
3. 登録後、案件名を押して詳細画面を開きます。

案件がないとAI社員チャットを使えません。最初に必ず案件を1件以上登録してください。

### 案件詳細

AIが回答に使う前提情報を設定します。

主に入力する内容は以下です。

- ブランド情報
- ブランドトーン
- ターゲット情報
- 必須訴求
- NG表現
- AI社員名
- AI社員の役割

ここを詳しく入力するほど、AIの投稿案が実務に近くなります。

### AI社員チャット

案件ごとにAI社員へ相談する画面です。

1. 左メニューの `AI社員チャット` を押します。
2. 左側の案件一覧から案件を選びます。
3. 入力欄に依頼内容を書きます。
4. 送信します。
5. AIの返信が良ければ `投稿案として保存` を押します。

例:

```text
30代女性向けに、今月のキャンペーン告知投稿を3案作ってください。
```

```text
この投稿案をフィード用に、冒頭の掴みを強くして書き直してください。
```

改行したい場合は、入力欄内で改行してから送信できます。

画像を見てもらいたい場合:

1. 入力欄左側のクリップアイコンを押します。
2. JPEG、PNG、WebPの画像を1枚選びます。
3. 必要に応じて依頼文を入力します。
4. `送信` を押します。

画像は2MB以下にしてください。AIは添付画像を確認し、投稿文、改善点、確認事項、次に行うタスクを提案できます。

例:

```text
この商品写真に合うフィード投稿文を作ってください。
```

```text
この画像をInstagram投稿に使う場合の改善点を教えてください。
```

### 投稿一覧

保存した投稿案を確認する画面です。

1. 左メニューの `投稿一覧` を押します。
2. 投稿タイトルを押します。
3. 投稿詳細で本文、形式、ステータス、予定日時を編集します。

投稿案が表示されない場合は、AI社員チャットのAI返信から `投稿案として保存` を押しているか確認してください。

### 投稿詳細

投稿ごとの管理画面です。

主に使う操作は以下です。

- 投稿内容の編集
- 投稿形式の確認
- ステータス変更
- 投稿予定日時の設定
- 承認依頼
- タスク抽出
- 投稿画像生成
- 実績入力画面への流れ作り

ステータスを `承認待ち` にすると、`承認フロー` に表示されます。

### 投稿カレンダー

投稿予定日時が入っている投稿を月間カレンダーで確認する画面です。

1. 投稿詳細で投稿予定日時を保存します。
2. 左メニューの `投稿カレンダー` を押します。
3. 月ごとの投稿予定を確認します。

予定日時が未設定の投稿はカレンダーに表示されません。

### 承認フロー

承認待ちの投稿と、最近の承認結果を確認する画面です。

1. 左メニューの `承認フロー` を押します。
2. `承認待ち` の投稿を開きます。
3. 投稿詳細で内容を確認します。
4. 承認または差し戻しを行います。

承認後は、投稿予約や公開作業へ進みます。

### タスク管理

投稿制作、確認、予約、公開後入力などの作業を管理する画面です。

1. 左メニューの `タスク管理` を押します。
2. 案件、投稿、担当内容、期限を入力してタスクを作成します。
3. 作業が終わったらステータスを更新します。

投稿詳細のタスク抽出機能を使うと、投稿案から必要作業をAIで抽出できます。

### 投稿実績入力

Instagramインサイトの数値を投稿ごとに保存する画面です。

1. 左メニューの `投稿実績入力` を押します。
2. 対象投稿を選びます。
3. 表示回数、リーチ、いいね、保存、コメントなどの数値を入力します。
4. 保存します。
5. 実績分析AIで改善提案を確認します。

現在は手入力です。Instagram Graph API連携を追加すると、将来的に自動取得も可能です。

### 月次レポート

入力済みの実績をもとに、月間サマリー、改善点、翌月方針をAIが作る画面です。

1. 左メニューの `月次レポート` を押します。
2. 対象案件を選びます。
3. 対象月を選びます。
4. レポート生成を実行します。

実績入力が多いほど、改善提案の精度が上がります。

### 初期設定

環境変数やSupabase設定の確認用画面です。通常運用では頻繁に使いません。

## 6. 投稿ステータスの目安

投稿一覧や承認フローで使うステータスの意味です。

| ステータス | 意味 |
| --- | --- |
| アイデア | まだ検討段階 |
| 企画中 | 投稿内容を作成中 |
| 事務確認中 | 社内確認中 |
| 承認待ち | 承認者の確認待ち |
| 承認済み | 投稿してよい状態 |
| 投稿予約済み | Instagram側で予約済み |
| 投稿済み | 実際に公開済み |
| 保留 | 一時停止 |
| 却下 | 採用しない投稿 |

## 7. AIに依頼するときのコツ

AI社員チャットでは、以下を入れると精度が上がります。

- 誰向けの投稿か
- 投稿形式
- 目的
- 必ず入れたい内容
- 避けたい表現
- 希望するトーン
- 文字量

例:

```text
美容サロンの30代女性向けに、フィード投稿の文章を作ってください。
目的は初回予約の増加です。
やさしく信頼感のあるトーンで、冒頭に悩みへの共感を入れてください。
```

## 8. よくあるトラブル

### ログインできない

Supabaseの `Authentication > Users` にユーザーが存在するか確認してください。パスワードが不明な場合は、Supabase側でユーザーを作り直すか、パスワードを再設定してください。

### 案件がAI社員チャットに表示されない

`案件一覧` で案件登録が完了しているか確認してください。登録直後に表示されない場合はページを再読み込みしてください。

### 投稿案が投稿一覧に表示されない

AI社員チャットのAI返信にある `投稿案として保存` を押してください。チャットしただけでは投稿一覧には入りません。

### カレンダーに投稿が表示されない

投稿詳細で投稿予定日時を入力してください。予定日時がない投稿はカレンダーに表示されません。

### AI生成に失敗する

Vercelの環境変数 `OPENAI_API_KEY` が正しいか、OpenAIの課金が有効か確認してください。

### 画像生成に失敗する

Vercelの環境変数を確認してください。

```text
OPENAI_IMAGE_MODEL=gpt-image-1
OPENAI_IMAGE_QUALITY=high
```

### 設定を変えたのに反映されない

Vercelの環境変数を変更した後は、必ずRedeployしてください。

```text
Vercel
→ Deployments
→ 最新Deploymentの ...
→ Redeploy
```

## 9. 管理者向けチェックリスト

新しい利用者を追加するとき:

- Supabaseでユーザーを追加した
- メールアドレスとパスワードを共有した
- Vercel公開URLを共有した
- 必要な案件が登録されている

新しい案件を開始するとき:

- 案件を作成した
- ブランド情報を入力した
- ターゲット情報を入力した
- NG表現を入力した
- AI社員設定を入力した
- AI社員チャットでテスト投稿案を作成した

毎月の運用締め:

- 投稿済みステータスを更新した
- 投稿実績を入力した
- 月次レポートを生成した
- 翌月の改善方針を確認した
