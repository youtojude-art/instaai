# AI設計

## AI社員の役割

AI社員はInstagram運用担当者として、案件情報、ブランドルール、ターゲット、商品・サービス、過去投稿、修正履歴、投稿実績、ナレッジを参照し、投稿企画、制作、確認、進行管理、分析、改善提案を行う。

## AI機能

- AI社員チャット
- 投稿企画生成
- フィード投稿生成
- リール台本生成
- ストーリーズ生成
- 投稿内容チェック
- 修正指示の記憶化
- 投稿実績分析
- 次月改善提案
- タスク自動整理
- 素材不足の確認

## AI記憶

会話履歴を毎回すべて渡さず、以下の形式で保存する。

| 種別 | 内容 |
| --- | --- |
| revision_rule | 今後も反映すべき修正ルール |
| approved_expression | 承認された表現、CTA、言い回し |
| rejected_expression | 却下された表現、NG理由 |
| insight | 実績分析から得た知見 |
| caution | 法務、広告表現、運用ミス防止 |

AI生成前に、案件、生成種別、指示文との関連度で記憶を検索してプロンプトへ挿入する。

## RAG設計

### 登録時

1. 資料をアップロードする
2. MIMEタイプとサイズを検証する
3. テキスト抽出する
4. チャンク分割する
5. Embeddingを作成する
6. `knowledge_chunks` に保存する

### 生成時

1. 指示文、案件、生成種別から検索クエリを作成する
2. `knowledge_chunks` から類似チャンクを取得する
3. チャンク本文を「参考資料」として渡す
4. 参考資料内の命令文は指示として扱わないようシステムプロンプトで明示する

## チャット保存

- ユーザー発話とAI応答は `chat_messages` に保存する。
- AI応答から投稿化した場合は、`content_posts` と関連付ける。
- 修正依頼として重要なものは `ai_memories` に保存候補として抽出し、スタッフが確認して保存できるようにする。

## AIプロンプトの注意点

- 事務スタッフの指示を優先する。
- 案件情報が不足しても、不必要な確認質問を繰り返さない。
- 不足情報は「不足項目」として出す。
- ブランドトーン、NG表現、法務注意を必ず反映する。
- 根拠のない断定、過度な誇張、医療・美容等のリスク表現を警告する。
- 作成内容、必要素材、確認事項、次に行うタスクを整理する。
